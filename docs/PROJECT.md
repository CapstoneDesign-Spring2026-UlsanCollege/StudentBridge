## Project Title

StudentBridge

## Problem Statement

International students in South Korea often struggle to find part-time jobs because job information is scattered, language requirements are unclear, and many general job platforms are not designed for student constraints. Local employers also need a simpler way to reach international student workers near universities.

## Target Users

- International students studying in South Korea.
- Local employers hiring part-time student workers.
- University support staff who help students find employment resources.

## Project Goal

StudentBridge will help students search student-friendly part-time jobs, create an account, and eventually submit job applications through one platform. The final capstone demo should show a small but reliable student flow supported by Java Servlets, Apache Tomcat, and MySQL.

## Core Features

- Homepage and navigation.
- Searchable job listings.
- Student/employer registration.
- Login through Java Servlet and MySQL.
- Job application submission.
- Basic employer job posting if time allows.

## Demo Scenario

1. Student opens StudentBridge.
2. Student searches jobs by title, city, category, or type.
3. Student registers or logs in.
4. Student selects a job and submits an application.
5. Team shows the stored database result or documented next-step blocker.

## Minimum Viable Product

The MVP is a working student job-search flow:

- search jobs,
- register/login,
- submit an application,
- store the result in MySQL,
- and explain the code and tests with linked evidence.

## Stretch Goals

- Employer dashboard.
- Korean/English language support.
- Resume upload.
- Email notifications.
- Public cloud deployment.
- AI job recommendations.
